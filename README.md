# Пробинг трансформерных языковых моделей на текстах Русского учебного корпуса (RLC)

Выпускная квалификационная работа по майнору «Нейросетевые технологии»
НИУ ВШЭ, 2026

Исследование посвящено тому, как родной язык (L1) учащегося и его уровень владения русским языком влияют на кодирование морфологической и синтаксической информации во внутренних представлениях трёх трансформерных моделей: **ruBERT**, **mBERT** и **XLM-RoBERTa**.

---

## Содержание репозитория

| Файл / папка | Назначение |
|---|---|
| `dataset_RLC_final_project.csv` | Исходный корпус (Russian Learner Corpus), отфильтрованный и подготовленный для анализа |
| `prepare_data.py` | Токенизация и морфосинтаксическая разметка (Stanza) |
| `extract_embeddings.py` | Извлечение скрытых представлений (слои 0–12) трёх моделей |
| `probing.py` | Пробинг-классификаторы по всем морфологическим и синтаксическим задачам |
| `probing_by_level.py` | Пробинг с группировкой по уровням CEFR (A1–C1), per-L1 и pooled |
| `dataset_analysis.py` | EDA датасета: распределение L1, длина текстов, TTR и т.д. |
| `results/` | CSV с результатами пробинга (используются в визуализациях) |
| `майнор_final_probing.pdf` | Полный текст работы |

### Выходные файлы по шагам

- `prepare_data.py` → `annotated_tokens.csv` (токены с морфологическими признаками), `annotated_sents.csv` (предложения с синтаксическими метками)
- `extract_embeddings.py` → папка `embeddings/`: `.npy`-файлы эмбеддингов + CSV-индексы
- `probing.py` → `results/probing_results.csv` (accuracy, selectivity, delta, expected layer)
- `probing_by_level.py` → `results/probing_bylevel_perlang.csv`, `results/probing_bylevel_pooled.csv`

## Запуск пайплайна

Шаги выполняются последовательно — каждый зависит от выхода предыдущего.

### 1. Подготовка данных

```bash
python prepare_data.py
```

> Убедитесь, что `dataset_RLC_final_project.csv` находится в той же папке, что и скрипт.

### 2. Извлечение эмбеддингов

```bash
python extract_embeddings.py
```

Самый долгий шаг — обрабатываются три модели по всем текстам. Для быстрой проверки можно ограничить число текстов и/или список моделей (см. комментарии в начале файла).

### 3. Пробинг

```bash
python probing.py
python probing_by_level.py
```

### 4. Визуализация (EDA)

```bash
python dataset_analysis.py
```

Графики сохраняются в текущую папку.

## Основные результаты

- **Морфология** (падеж, число, вид, время, род, лицо, наклонение) кодируется всеми тремя моделями на нижних и средних слоях (3–6), и эта закономерность устойчива независимо от типологии L1.
- **Синтаксис** (позиция подлежащего, порядок слов) чувствителен к родному языку: носители SOV-языков (корейский, казахский) хуже предсказывают канонический SVO-порядок в русском.
- **Уровень владения языком** почти не влияет на качество кодирования морфологии, но для синтаксиса наблюдается монотонное снижение точности от A1 к C1 — по мере роста уровня тексты становятся синтаксически разнообразнее.
- **Среди моделей** XLM-RoBERTa стабильно превосходит ruBERT и mBERT, особенно на задачах с нестандартными морфологическими формами.

Подробное описание методологии, экспериментов и интерпретации — в `майнор_final_probing.pdf`.
